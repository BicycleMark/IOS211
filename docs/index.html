Before you begin

This is an iOS class. To complete the coding exercises, you will need a Xamarin.iOS development environment with either a configured iOS Simulator or a device to run the exercise on. Please read the setup requirements if you need help installing the Xamarin development environment.

Overview

After this session you will be able to improve the user experience of your application by allowing an ongoing operation to continue, even if your app is no longer the active one.

Using the Simulator to run the lab exercises

Some of the lab exercises for this class utilize AVAudioSession to play music, both while the app is running and while it is not. If you plan to run the labs on the iOS Simulator, you will need to make sure you have both a valid input (microphone) and output (speaker) selected in the Audio settings on your Mac. If either of these are missing, the AVAudioSession will fail to initialize - the simulator requests both input and output on every session even if you only request output support. Some Macs, such as the Mac Mini do not include a built-in microphone and in this case you will need to plug in an external microphone such as a Mic/Headphone combination to get the labs to work properly.

Objectives

  1. Perform Long-Running Tasks without time limits
  2. Transfer files in the background

Lecture

IOS211 - Background Modes and File Transfers

Exercises

Exercise materials are available from GitHub. The code can be cloned through a desktop Git client or downloaded directly as a ZIP file.

  1. Play music in the background
    Exercise
    You will start music playback, keep it running if the app is backgrounded and react to remote control events.
  2. Download a file in the background
    Exercise
    You will use NSUrlSession to start a file download, hand it off to the operating system and keep it going even if the app gets terminated.

Additional resources

The lab materials contain a more complex demo that downloads music in the background, plays it back and shows how to serialize the current app state between restarts of the app. You can find it in the Bonus folder.
